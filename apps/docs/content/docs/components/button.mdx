## Overview

The Button component is a fundamental building block for user interactions. It supports multiple variants, sizes, and states to fit various design needs.

## Installation

```bash
npm install @px-ui/core
```

## Import

```tsx
import { Button } from '@px-ui/core';
```

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `variant` | `'default' \| 'primary' \| 'destructive' \| 'outline' \| 'primary-outline' \| 'ghost' \| 'link'` | `'default'` | The visual style variant of the button |
| `size` | `'sm' \| 'default' \| 'lg' \| 'icon-sm' \| 'icon'` | `'default'` | The size of the button |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `className` | `string` | - | Additional CSS classes to apply |
| ...rest | `React.ComponentProps<'button'>` | - | All standard HTML button attributes |

## Accessibility

- The Button component is rendered as a native `<button>` element, ensuring proper keyboard navigation and screen reader support.
- Disabled buttons automatically receive `cursor-not-allowed` styling.
- Focus states are clearly indicated with a focus ring (`focus-visible:ring-3`).
- Use descriptive text or `aria-label` for icon-only buttons to ensure screen reader accessibility.

## Examples

### Loading State

You can compose the Button with a spinner for loading states:

```tsx
import { Button } from '@px-ui/core';
import { Spinner } from '@px-ui/core';

<Button disabled>
  <Spinner size="sm" />
  Loading...
</Button>
```

### As Link

While there's a `link` variant, you can also use the button styling with other elements:

```tsx
<a
  href="/somewhere"
  className={buttonVariants({ variant: "primary" })}
>
  Navigate
</a>
```

## Best Practices

- **Use Primary for main actions**: Reserve the `primary` variant for the most important action on a page.
- **Use Destructive carefully**: Only use the `destructive` variant for actions that delete or remove data.
- **Icon buttons need labels**: Always provide accessible labels for icon-only buttons using `aria-label`.
- **Don't overuse**: Too many buttons can overwhelm users. Consider using links or other interactive elements when appropriate.
