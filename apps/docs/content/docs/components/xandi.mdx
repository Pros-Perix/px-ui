## Overview

Xandi is a composable AI chatbot component designed for workforce and hiring applications. It provides a ready-to-use chat interface with welcome messages, prompt suggestions, message actions, and chat history management.

## Import

```tsx
import { Xandi, XandiProvider, XHeader, XSidebar } from "@px-ui/ai";
```

## Architecture

Xandi uses a provider pattern to manage chat state. The `XandiProvider` wraps your chat components and provides context for:

- **Message state** - Manages conversation history
- **Session management** - Handles chat sessions with optional session restoration
- **API communication** - Delegates to your custom fetch function

### Basic Setup

```tsx
import { type XandiResponse } from "@px-ui/ai";

async function fetchResponse(message: string): Promise<XandiResponse> {
  const response = await fetch("https://api.example.com/chat", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "Authorization": "Bearer your-token",
    },
    body: JSON.stringify({ message }),
  });
  const data = await response.json();
  return {
    content: data.response,
    debugTrace: data.debug, // optional
  };
}

<XandiProvider
  fetchResponse={fetchResponse}
  sessionId="existing-session-id" // optional, for restoring sessions
  onFeedback={(messageId, feedback) => console.log(messageId, feedback)}
>
  <Xandi
    welcomeMessage="How can I help you today?"
    suggestions={[
      { id: "1", label: "Open Jobs", prompt: "Show me all open jobs" },
      { id: "2", label: "Pending Timesheets", prompt: "Show me pending timesheets" },
    ]}
  />
</XandiProvider>
```

### With Header and Sidebar

```tsx
import { useState } from "react";

    body: JSON.stringify({ message }),
  });
  const data = await response.json();
  return { content: data.response };
}

function ChatWindow() {
  const [sidebarOpen, setSidebarOpen] = useState(false);

  return (
    <div className="flex h-full">
      <XSidebar
        isOpen={sidebarOpen}
        chatHistory={chatHistory}
```

## API Reference

### XandiProvider

The context provider that manages chat state. Requires a `fetchResponse` function that handles the API transaction.

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `fetchResponse` | `(message: string) => Promise<XandiResponse>` | Yes | Function that sends a message and returns the AI response |
| `sessionId` | `string` | No | Existing session ID to restore a chat |
| `onFeedback` | `(messageId: string, feedback: FeedbackType) => void` | No | Callback when user provides feedback on a message |

### Xandi

The main chat component. Must be wrapped in `XandiProvider`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `welcomeMessage` | `string` | `"How can I help you today?"` | Message shown when chat is empty |
| `suggestions` | `Suggestion[]` | `[]` | Prompt suggestions shown below the input |

### XHeader

Header component for floating windows and sidebars.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | `"Xandi"` | Title displayed in the header |
| `onClose` | `() => void` | - | Callback when close button is clicked |
| `onNewChat` | `() => void` | - | Callback when new chat button is clicked |
| `onToggleHistory` | `() => void` | - | Callback when menu button is clicked |

### XSidebar

Sidebar component for chat history.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `isOpen` | `boolean` | `true` | Whether the sidebar is visible |
| `chatHistory` | `ChatHistoryGroup[]` | `[]` | Grouped chat history items |
| `activeChatId` | `string` | - | ID of the currently active chat |
| `onClose` | `() => void` | - | Callback when close button is clicked |
| `onNewChat` | `() => void` | - | Callback when new chat button is clicked |
| `onSelectChat` | `(chatId: string) => void` | - | Callback when a chat is selected |

### useXandi Hook

Access the chat context from any component within `XandiProvider`.

```tsx
import { useXandi } from "@px-ui/ai";

function CustomComponent() {
  const { messages, isLoading, sessionId, sendMessage } = useXandi();
  // ...
}
```

| Property | Type | Description |
|----------|------|-------------|
| `messages` | `Message[]` | Array of chat messages |
| `isLoading` | `boolean` | Whether a response is being fetched |
| `sessionId` | `string \| null` | Current session identifier |
| `sendMessage` | `(text: string) => void` | Function to send a message |
| `onFeedback` | `(messageId: string, feedback: FeedbackType) => void` | Feedback handler from provider |

## Types

```tsx
interface XandiResponse {
  content: string;
  type?: "text" | "markdown";
  debugTrace?: unknown;
}

interface Suggestion {
  id: string;
  label: string;
  prompt: string;
}

interface Message {
  id: string;
  role: "user" | "assistant";
  content: string;
  type?: "text" | "markdown";
  debugTrace?: unknown;
}

type FeedbackType = "up" | "down" | null;

interface ChatHistoryItem {
  id: string;
  title: string;
  timestamp: Date;
}

interface ChatHistoryGroup {
  label: string;
  items: ChatHistoryItem[];
}
```

## Message Actions

Assistant messages include action buttons for feedback, copying, and debugging.

- **Thumbs Up/Down** - Provide feedback on message quality
- **Copy** - Copy message content to clipboard
- **Debug** - View debug trace (when `debugTrace` is present in the message)
