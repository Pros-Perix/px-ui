## Overview

The Form components provide convenient abstractions over react-hook-form's Controller for common form fields. They automatically handle field registration, validation, and error display.

## Import

```tsx
import { FormInput, FormTextarea, FormCheckbox } from "@px-ui/forms";
```

## Usage

<Preview>
  <form onSubmit={(e) => { e.preventDefault(); console.log("Submitted"); }}>
    <FormInput
      name="username"
      label="Username"
      control={control}
      placeholder="Enter your username"
    />
  </form>
</Preview>

<CodeBlock>
```tsx
import { useForm } from "react-hook-form";
import { FormInput } from "@px-ui/forms";

function MyForm() {
  const { control } = useForm();
  
  return (
    <form>
      <FormInput
        name="username"
        label="Username"
        control={control}
        placeholder="Enter your username"
      />
    </form>
  );
}
```
</CodeBlock>

## Props

### FormInput

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | - | Field name (must match form schema) |
| `label` | `ReactNode` | - | Label text |
| `control` | `Control` | - | react-hook-form control object |
| `description` | `ReactNode` | - | Description text |
| `required` | `boolean` | - | Mark field as required |
| `horizontal` | `boolean` | `false` | Use horizontal layout |
| `controlFirst` | `boolean` | `false` | Render control before label |
| `type` | `string` | `"text"` | Input type |
| `placeholder` | `string` | - | Placeholder text |
| `size` | `"default" \| "sm"` | - | Size of the input |
| `widthVariant` | `"enforced" \| "full"` | - | Width behavior |
| `disabled` | `boolean` | - | Disable the input |
| `rules` | `object` | - | react-hook-form validation rules |
| `className` | `string` | - | Additional CSS classes |

Plus all standard Input component props.
